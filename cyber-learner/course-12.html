Burp Suite

Master web application security testing with Burp Suite

Introduction to Burp Suite

Burp Suite is the leading web application security testing platform. It's an integrated platform for performing security testing of web applications, combining manual and automated techniques.

What is Burp Suite?

  • Web Proxy: Intercept and modify HTTP/HTTPS traffic
  • Scanner: Automated vulnerability detection
  • Intruder: Automated customized attacks
  • Repeater: Manual request manipulation and testing
  • Sequencer: Analyze randomness of session tokens

Burp Suite Editions

Community Edition

  • Free version
  • Basic proxy functionality
  • Limited scanner features
  • Manual testing tools

Professional Edition

  • Full-featured scanner
  • Advanced tools
  • Extensive reporting
  • API integration

Enterprise Edition

  • Continuous scanning
  • CI/CD integration
  • Team collaboration
  • Enterprise reporting

Getting Started

  1. Start Burp Suite: Launch the application
  2. Create Project: Set up temporary or disk-based project
  3. Configure Browser: Set proxy settings
  4. Install Certificate: Add Burp's CA certificate
  5. Define Scope: Set target scope for testing

Proxy Configuration

The Proxy tool is the heart of Burp Suite, allowing you to intercept and modify web traffic.

# Proxy Configuration Steps:

1. Burp Proxy Settings:
   - Default: 127.0.0.1:8080
   - Enable "Intercept is on"
   - Configure SSL/TLS settings

2. Browser Configuration:
   - Set HTTP proxy: 127.0.0.1:8080
   - Set HTTPS proxy: 127.0.0.1:8080
   - Or use browser extensions like FoxyProxy

3. Certificate Installation:
   - Browse to http://burp
   - Download CA Certificate
   - Install in browser's certificate store

4. Testing Setup:
   - Visit target website
   - Verify traffic appears in HTTP history
   - Check intercept functionality

# Proxy Options:
Intercept Client Requests    - Intercept requests from browser
Intercept Server Responses   - Intercept responses from server
Match and Replace           - Automatic modifications
SSL Pass Through           - Bypass SSL for specific hosts

Proxy Features

  • HTTP History: View all intercepted requests and responses
  • WebSockets History: Monitor WebSocket communications
  • Options: Configure proxy behavior and rules
  • Intercept: Manually inspect and modify traffic

Target Tool

The Target tool helps organize and scope your testing activities.

# Target Tool Features:

Site Map:
- Hierarchical view of discovered content
- Shows all URLs, parameters, and forms
- Color-coded by response status
- Expandable tree structure

Scope:
- Define which hosts/URLs to test
- Include/exclude specific paths
- Use with other tools for focused testing
- Prevent testing out-of-scope targets

# Setting Up Target Scope:
1. Browse target application
2. Right-click on target in Site Map
3. Select "Add to scope"
4. Configure include/exclude rules
5. Enable "Show only in-scope items"

# Site Map Information:
- Request/Response details
- Parameters and their values
- Comments and highlights
- Issue annotations
- Engagement tools integration

Site Map Analysis

  • Content Discovery: Identify all application endpoints
  • Parameter Analysis: Find input points for testing
  • Technology Stack: Identify frameworks and technologies
  • Attack Surface: Map potential vulnerability locations

Scanner Tool

Burp Scanner automatically identifies security vulnerabilities in web applications.

# Scanner Capabilities:

Vulnerability Types:
- SQL Injection
- Cross-Site Scripting (XSS)
- Cross-Site Request Forgery (CSRF)
- XML External Entity (XXE)
- Server-Side Request Forgery (SSRF)
- Directory Traversal
- Command Injection
- Authentication Bypass
- Session Management Issues
- Information Disclosure

# Scanning Methods:
Active Scanning:
- Sends crafted requests to find vulnerabilities
- May cause side effects on target application
- More thorough but potentially disruptive

Passive Scanning:
- Analyzes existing traffic for issues
- No additional requests sent
- Safe but limited coverage

# Scanner Configuration:
1. Select target items to scan
2. Choose scan type (active/passive)
3. Configure scan settings
4. Monitor scan progress
5. Review identified issues

Scan Results

High Severity

  • SQL Injection
  • Remote Code Execution
  • Authentication Bypass
  • Privilege Escalation

Medium Severity

  • Cross-Site Scripting
  • CSRF vulnerabilities
  • Directory Traversal
  • Information Disclosure

Low Severity

  • Missing security headers
  • Weak SSL configuration
  • Information leakage
  • Cookie security issues

Intruder Tool

Intruder is Burp's automated attack tool for customized attacks against web applications.

# Intruder Attack Types:

Sniper:
- Single payload set
- Tests one parameter at a time
- Good for focused testing

Battering Ram:
- Single payload set
- Same payload in all positions
- Useful for password attacks

Pitchfork:
- Multiple payload sets
- Iterates through payloads in parallel
- Good for username/password combinations

Cluster Bomb:
- Multiple payload sets
- Tests all combinations
- Most comprehensive but slowest

# Intruder Workflow:
1. Send request to Intruder
2. Set attack positions (¬ßpayload¬ß)
3. Choose attack type
4. Configure payload sets
5. Set processing rules
6. Start attack
7. Analyze results

Payload Types

  • Simple List: Predefined wordlists
  • Runtime File: Read from external file
  • Numbers: Sequential or random numbers
  • Dates: Date ranges and formats
  • Brute Forcer: Character set combinations
  • Null Payloads: Empty payloads for timing attacks

Common Intruder Use Cases

# Password Brute Force:
1. Capture login request
2. Send to Intruder
3. Set password field as payload position
4. Load password wordlist
5. Configure attack settings
6. Analyze response lengths/status codes

# Directory Enumeration:
1. Capture directory request
2. Set directory name as payload position
3. Use directory wordlist
4. Look for 200 OK responses
5. Identify hidden directories

# Parameter Fuzzing:
1. Identify input parameters
2. Set parameters as payload positions
3. Use fuzzing wordlists
4. Monitor for errors or unusual responses
5. Identify potential vulnerabilities

# Repeater Features:

Manual Testing:
- Modify requests before sending
- View raw request/response data
- Test different parameter values
- Analyze application behavior

Request Manipulation:
- Change HTTP methods
- Modify headers
- Alter parameter values
- Add/remove parameters
- Test encoding variations

Response Analysis:
- Compare different responses
- Identify error messages
- Analyze response timing
- Check for information disclosure

# Common Repeater Workflows:

SQL Injection Testing:
1. Send request to Repeater
2. Add SQL injection payloads
3. Analyze error messages
4. Test different injection techniques
5. Confirm vulnerability

XSS Testing:
1. Identify input parameters
2. Insert XSS payloads
3. Check if payload is reflected
4. Test different encoding methods
5. Verify script execution

Repeater Best Practices

  • Systematic Testing: Test one parameter at a time
  • Payload Variations: Try different encoding and formats
  • Response Comparison: Compare normal vs. malicious responses
  • Documentation: Keep notes of successful payloads

Sequencer Tool

Sequencer analyzes the randomness and predictability of session tokens and other important data items.

# Sequencer Analysis:

Token Collection:
- Capture session tokens automatically
- Collect large sample sizes (1000+ tokens)
- Analyze different token types
- Test various application states

Randomness Tests:
- FIPS 140-2 statistical tests
- Entropy analysis
- Pattern detection
- Correlation analysis
- Compression analysis

# Sequencer Workflow:
1. Identify token generation request
2. Send to Sequencer
3. Configure token extraction
4. Start live capture
5. Collect sufficient samples
6. Analyze results
7. Generate report

# Token Analysis Results:
Overall Quality: Excellent/Good/Poor
Effective Entropy: Bits of randomness
Statistical Tests: Pass/Fail status
Character-level Analysis: Distribution patterns
Bit-level Analysis: Randomness patterns

Common Token Issues

  • Low Entropy: Predictable token generation
  • Time-based Patterns: Tokens based on timestamps
  • Sequential Patterns: Incrementing token values
  • Weak Algorithms: Poor random number generation

# Decoder Functions:

Encoding Types:

# Decoder Workflow:

# Comparer Functions:

Response Comparison:

- Identify authentication bypass attempts
- Analyze error message variations
- Detect information disclosure differences

Practical Testing Scenarios

SQL Injection Testing

# SQL Injection Testing Process:

1. Identify Input Points:
   - Form parameters
   - URL parameters
   - HTTP headers
   - Cookie values

2. Test for SQL Injection:
   - Add single quotes (')
   - Use SQL injection payloads
   - Analyze error messages
   - Test different injection techniques

3. Common Payloads:
   ' OR '1'='1
   ' UNION SELECT NULL--
   '; DROP TABLE users--
   ' AND 1=1--
   ' AND 1=2--

4. Burp Tools for SQLi:
   - Scanner: Automated detection
   - Intruder: Payload testing
   - Repeater: Manual verification
   - Extensions: SQLMap integration

Cross-Site Scripting (XSS) Testing

# XSS Testing Methodology:

1. Identify Reflection Points:
   - Search parameters
   - Form inputs
   - URL parameters
   - Error messages

2. Test XSS Payloads:
   <script>alert('XSS')</script>
   <img src=x onerror=alert('XSS')>
   javascript:alert('XSS')
   <svg onload=alert('XSS')>

3. Bypass Filters:
   - Encoding variations
   - Case manipulation
   - Tag variations
   - Event handler alternatives

4. Burp XSS Testing:
   - Scanner: Automated XSS detection
   - Intruder: Payload fuzzing
   - Repeater: Manual testing
   - Extensions: XSS validation

Best Practices and Tips

Testing Methodology

  • Scope Definition: Clearly define testing boundaries
  • Systematic Approach: Follow a structured testing methodology
  • Documentation: Keep detailed notes and screenshots
  • Verification: Confirm all findings manually
  • Reporting: Provide clear, actionable reports

Performance Optimization

  • Scope Limiting: Test only necessary components
  • Thread Management: Adjust concurrent requests
  • Memory Settings: Allocate sufficient memory
  • Selective Scanning: Choose appropriate scan types

Common Pitfalls

  • Over-reliance on Automation: Always verify automated findings
  • Scope Creep: Stay within authorized testing boundaries
  • False Positives: Validate all scanner results
  • Incomplete Testing: Test all application functionality
