Nmap

Master network discovery and security auditing with Nmap

Introduction to Nmap

Nmap (Network Mapper) is a powerful open-source tool for network discovery and security auditing. It's essential for penetration testers, network administrators, and security professionals.

What is Nmap?

  • Network Discovery: Find live hosts on networks
  • Port Scanning: Identify open ports and services
  • OS Detection: Determine operating systems
  • Service Enumeration: Identify service versions
  • Vulnerability Detection: Find security weaknesses

Basic Nmap Usage

Understanding basic Nmap syntax and common scanning techniques.

# Basic Nmap Syntax:
nmap [Scan Type] [Options] [Target]

# Simple Host Discovery:
nmap 192.168.1.1                    # Scan single host
nmap 192.168.1.1-254                # Scan IP range
nmap 192.168.1.0/24                 # Scan subnet
nmap scanme.nmap.org                # Scan hostname

# Multiple Targets:
nmap 192.168.1.1 192.168.1.5        # Multiple IPs
nmap -iL targets.txt                # Read from file
nmap 192.168.1.* --exclude 192.168.1.1  # Exclude hosts

# Common Options:
-v      # Verbose output
-vv     # Very verbose
-n      # No DNS resolution
-Pn     # Skip host discovery
-6      # IPv6 scanning

Target Specification

Single Host

  • nmap 192.168.1.1
  • nmap example.com
  • Most basic scan type

IP Range

  • nmap 192.168.1.1-100
  • nmap 192.168.1-3.1
  • Scan consecutive IPs

Subnet/CIDR

  • nmap 192.168.1.0/24
  • nmap 10.0.0.0/8
  • Scan entire networks

Host Discovery Techniques

Nmap uses various methods to determine if hosts are alive before port scanning.

# Host Discovery Options:

# Ping Scan (Default):
nmap -sn 192.168.1.0/24             # Ping sweep only
nmap -sP 192.168.1.0/24             # Legacy ping scan

# ICMP Discovery:
nmap -PE 192.168.1.0/24             # ICMP Echo
nmap -PP 192.168.1.0/24             # ICMP Timestamp
nmap -PM 192.168.1.0/24             # ICMP Address Mask

# TCP Discovery:
nmap -PS22,80,443 192.168.1.0/24    # TCP SYN ping
nmap -PA22,80,443 192.168.1.0/24    # TCP ACK ping

# UDP Discovery:
nmap -PU53,161,137 192.168.1.0/24   # UDP ping

# ARP Discovery (Local Network):
nmap -PR 192.168.1.0/24             # ARP ping

# Skip Host Discovery:
nmap -Pn 192.168.1.1                # Treat all hosts as online

Discovery Method Selection

  • Local Network: ARP ping is most reliable
  • Internet Hosts: TCP SYN ping to common ports
  • Firewalled Hosts: Multiple discovery methods
  • Stealth Scanning: Skip discovery with -Pn

Port Scanning Techniques

Nmap offers various port scanning methods, each with different characteristics.

TCP Connect Scan (-sT)

  • Completes full TCP handshake
  • Most reliable scan type
  • Easily detected and logged
  • Default for non-root users

SYN Scan (-sS)

  • Half-open scanning
  • Stealthy and fast
  • Requires root privileges
  • Default for root users

UDP Scan (-sU)

  • Scans UDP ports
  • Slower than TCP scans
  • Often filtered by firewalls
  • Important for DNS, DHCP, SNMP
# Port Scanning Examples:

# TCP Scans:
nmap -sS 192.168.1.1                # SYN scan (stealth)
nmap -sT 192.168.1.1                # TCP connect scan
nmap -sA 192.168.1.1                # ACK scan (firewall detection)
nmap -sW 192.168.1.1                # Window scan
nmap -sF 192.168.1.1                # FIN scan
nmap -sX 192.168.1.1                # Xmas scan
nmap -sN 192.168.1.1                # Null scan

# UDP Scan:
nmap -sU 192.168.1.1                # UDP scan
nmap -sS -sU 192.168.1.1            # TCP + UDP scan

# Port Specification:
nmap -p 80 192.168.1.1              # Single port
nmap -p 80,443,22 192.168.1.1       # Multiple ports
nmap -p 1-1000 192.168.1.1          # Port range
nmap -p- 192.168.1.1                # All ports (1-65535)
nmap --top-ports 100 192.168.1.1    # Top 100 ports

Advanced Scanning Options

# Timing and Performance:
nmap -T0 192.168.1.1                # Paranoid (slowest)
nmap -T1 192.168.1.1                # Sneaky
nmap -T2 192.168.1.1                # Polite
nmap -T3 192.168.1.1                # Normal (default)
nmap -T4 192.168.1.1                # Aggressive
nmap -T5 192.168.1.1                # Insane (fastest)

# Custom Timing:
nmap --min-rate 1000 192.168.1.1    # Minimum packet rate
nmap --max-rate 5000 192.168.1.1    # Maximum packet rate
nmap --min-parallelism 10 192.168.1.1  # Parallel probes

# Firewall Evasion:
nmap -f 192.168.1.1                 # Fragment packets
nmap -D RND:10 192.168.1.1          # Decoy scanning
nmap -S 192.168.1.5 192.168.1.1     # Source IP spoofing
nmap --source-port 53 192.168.1.1   # Source port manipulation

Service and Version Detection

Nmap can identify services running on open ports and their versions.

# Service Detection:
nmap -sV 192.168.1.1                # Version detection
nmap -sV --version-intensity 5 192.168.1.1  # Intensity level (0-9)
nmap -sV --version-all 192.168.1.1  # Try all probes

# Service Detection Examples:
22/tcp   open  ssh     OpenSSH 7.4 (protocol 2.0)
80/tcp   open  http    Apache httpd 2.4.6
443/tcp  open  https   Apache httpd 2.4.6 ((CentOS) OpenSSL/1.0.2k)
3306/tcp open  mysql   MySQL 5.7.25

# Aggressive Service Detection:
nmap -A 192.168.1.1                 # OS + Version + Script + Traceroute
nmap -sV -O --script=default 192.168.1.1  # Equivalent to -A

# Service-Specific Scans:
nmap -p 80 --script http-* 192.168.1.1     # HTTP scripts
nmap -p 22 --script ssh-* 192.168.1.1      # SSH scripts
nmap -p 3306 --script mysql-* 192.168.1.1  # MySQL scripts

Version Detection Intensity

  • Level 0: Light probes only
  • Level 5: Default intensity
  • Level 9: All available probes
  • Higher levels: More accurate but slower

Operating System Detection

Nmap can identify target operating systems using various fingerprinting techniques.

# OS Detection:
nmap -O 192.168.1.1                 # Basic OS detection
nmap -O --osscan-guess 192.168.1.1  # Aggressive OS guessing
nmap -O --osscan-limit 192.168.1.1  # Limit to promising targets

# OS Detection Output Example:
Running: Linux 3.X|4.X
OS CPE: cpe:/o:linux:linux_kernel:3 cpe:/o:linux:linux_kernel:4
OS details: Linux 3.2 - 4.9
Network Distance: 1 hop

# Factors Affecting OS Detection:
- At least one open port required
- One closed port helpful
- Firewall interference
- Network distance
- Target OS variations

# OS Detection with Service Detection:
nmap -A 192.168.1.1                 # Complete system scan
nmap -O -sV 192.168.1.1             # OS + Version detection

OS Detection Techniques

  • TCP ISN Sampling: Initial sequence number patterns
  • TCP Options: Supported TCP options
  • IP ID Sampling: IP identification field behavior
  • ICMP Responses: ICMP error message analysis

Nmap Scripting Engine (NSE)

NSE extends Nmap's capabilities with hundreds of specialized scripts.

# Script Categories:
auth        # Authentication scripts
broadcast   # Network broadcast discovery
brute       # Brute force attacks
default     # Default safe scripts
discovery   # Network/service discovery
dos         # Denial of service
exploit     # Vulnerability exploitation
external    # External resource queries
fuzzer      # Fuzzing scripts
intrusive   # Intrusive scripts
malware     # Malware detection
safe        # Safe scripts
version     # Version detection enhancement
vuln        # Vulnerability detection

# Running Scripts:
nmap --script=default 192.168.1.1   # Default scripts
nmap --script=safe 192.168.1.1      # Safe scripts only
nmap --script=vuln 192.168.1.1      # Vulnerability scripts
nmap --script=auth 192.168.1.1      # Authentication scripts

# Specific Scripts:
nmap --script=http-title 192.168.1.1        # HTTP title
nmap --script=ssl-cert 192.168.1.1          # SSL certificate
nmap --script=smb-os-discovery 192.168.1.1  # SMB OS info
nmap --script=dns-brute example.com         # DNS brute force

Output Formats and Reporting

Nmap supports various output formats for different use cases.

# Output Formats:
nmap -oN scan.txt 192.168.1.1       # Normal output
nmap -oX scan.xml 192.168.1.1       # XML output
nmap -oG scan.gnmap 192.168.1.1     # Grepable output
nmap -oA scan 192.168.1.1           # All formats

# Output Examples:

# Normal Output (-oN):
Nmap scan report for 192.168.1.1
Host is up (0.00050s latency).
PORT   STATE SERVICE
22/tcp open  ssh
80/tcp open  http

# XML Output (-oX):
<?xml version="1.0" encoding="UTF-8"?>
<nmaprun scanner="nmap" version="7.80">
  <host>
    <address addr="192.168.1.1" addrtype="ipv4"/>
    <ports>
      <port protocol="tcp" portid="22">
        <state state="open"/>
      </port>
    </ports>
  </host>
</nmaprun>

# Grepable Output (-oG):
Host: 192.168.1.1 ()    Status: Up
Host: 192.168.1.1 ()    Ports: 22/open/tcp//ssh///, 80/open/tcp//http///

Advanced Output Options

  • --append-output: Append to existing files
  • --resume: Resume interrupted scans
  • --stylesheet: Custom XML stylesheet
  • --webxml: Reference Nmap.org stylesheet

Practical Nmap Examples

Network Discovery

# Discover live hosts on network:
nmap -sn 192.168.1.0/24

# Fast network sweep:
nmap -sn --min-rate 5000 192.168.1.0/24

# Comprehensive host discovery:
nmap -PE -PP -PS21,22,23,25,53,80,113,31339 -PA80,113,443,10042 192.168.1.0/24

Service Enumeration

# Quick service scan:
nmap -sS -O -sV --top-ports 1000 192.168.1.1

# Comprehensive service scan:
nmap -sS -sU -O -sV -p- 192.168.1.1

# Web server enumeration:
nmap -p 80,443 --script http-enum,http-headers,http-methods 192.168.1.1

Vulnerability Assessment

# Basic vulnerability scan:
nmap --script vuln 192.168.1.1

# Specific vulnerability checks:
nmap --script smb-vuln-* 192.168.1.1
nmap --script ssl-* 192.168.1.1

# Safe vulnerability assessment:
nmap --script "vuln and safe" 192.168.1.1

Nmap Best Practices

Ethical Considerations

  • Authorization: Only scan networks you own or have permission to test
  • Legal Compliance: Understand local laws and regulations
  • Responsible Disclosure: Report vulnerabilities appropriately
  • Documentation: Keep records of authorized testing

Performance Optimization

  • Target Selection: Scan only necessary hosts and ports
  • Timing Templates: Use appropriate timing for network conditions
  • Parallel Scanning: Adjust parallelism for better performance
  • Output Management: Use appropriate output formats

Stealth Considerations

  • Scan Timing: Spread scans over time
  • Source Randomization: Use decoys and source port manipulation
  • Fragmentation: Fragment packets to evade detection
  • Script Selection: Avoid intrusive scripts when stealth is important
