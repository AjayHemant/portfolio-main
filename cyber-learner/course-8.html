Packet Analysis

Learn to capture, analyze, and interpret network traffic

Introduction to Packet Analysis

Packet analysis is the process of capturing, examining, and interpreting network traffic. It's essential for network troubleshooting, security monitoring, and understanding network behavior.

Why Packet Analysis Matters

  • Network Troubleshooting: Identify connectivity issues
  • Security Monitoring: Detect malicious activity
  • Performance Analysis: Optimize network performance
  • Protocol Understanding: Learn how protocols work
  • Forensic Investigation: Analyze security incidents

- Reserved (6 bits)
- Flags (6 bits)
- Window Size (16 bits)
- Checksum (16 bits)
- Urgent Pointer (16 bits)
- Options (variable)

Packet Capture Methods

There are several ways to capture network packets for analysis.

Capture Techniques

Promiscuous Mode

  • Captures all packets on network segment
  • Requires special network card configuration
  • Works on hubs and some switches
  • Limited by switch port mirroring

Port Mirroring (SPAN)

  • Switch copies traffic to monitor port
  • Can mirror specific ports or VLANs
  • Requires switch configuration
  • Most common in enterprise networks

Network TAP

  • Physical device inserted in network path
  • Copies all traffic without modification
  • No impact on network performance
  • Most reliable capture method

Wireless Capture

  • Monitor mode on wireless adapters
  • Captures all wireless frames
  • Includes management and control frames
  • Requires compatible hardware

Common Analysis Tools

Various tools are available for packet capture and analysis.

Wireshark

# Wireshark Features:
- Graphical packet analyzer
- Supports 1000+ protocols
- Real-time and offline analysis
- Powerful filtering capabilities
- Cross-platform (Windows, Linux, macOS)

# Common Wireshark Filters:
ip.addr == 192.168.1.1          # Specific IP address
tcp.port == 80                  # HTTP traffic
http.request.method == "GET"    # HTTP GET requests
dns                             # DNS traffic only
tcp.flags.syn == 1              # TCP SYN packets
icmp                            # ICMP packets
not arp                         # Exclude ARP traffic

# Display Filter Examples:
tcp.stream eq 0                 # Follow TCP stream
frame.time >= "2023-01-01"      # Time-based filtering
tcp.analysis.retransmission     # Show retransmissions
http.response.code == 404       # HTTP 404 errors

Command Line Tools

# tcpdump (Linux/Unix):
tcpdump -i eth0                 # Capture on interface eth0
tcpdump -i eth0 -w capture.pcap # Write to file
tcpdump -r capture.pcap         # Read from file
tcpdump host 192.168.1.1        # Specific host
tcpdump port 80                 # Specific port
tcpdump -n -c 100               # No DNS resolution, 100 packets

# tshark (Wireshark command line):
tshark -i eth0                  # Live capture
tshark -r capture.pcap          # Read file
tshark -Y "http"                # Display filter
tshark -T fields -e ip.src      # Extract specific fields

# netstat (Connection information):
netstat -an                     # All connections
netstat -rn                     # Routing table
netstat -i                      # Interface statistics

Network Performance Analysis

Key Performance Metrics

  • Latency: Round-trip time for packets
  • Throughput: Data transfer rate
  • Packet Loss: Percentage of lost packets
  • Jitter: Variation in packet delay
  • Bandwidth Utilization: Network capacity usage

Performance Analysis Techniques

# Latency Analysis:
- Measure time between request and response
- Look for TCP handshake timing
- Analyze ping/ICMP response times
- Check for network congestion indicators

# Throughput Analysis:
- Calculate bytes per second
- Identify bottlenecks
- Analyze TCP window scaling
- Monitor buffer utilization

# Packet Loss Detection:
- Missing sequence numbers
- Duplicate ACKs
- Retransmission patterns
- TCP fast retransmit indicators

# Quality of Service (QoS) Analysis:
- DSCP markings in IP header
- Traffic prioritization
- Bandwidth allocation
- Queue management

Security Analysis

Packet analysis is crucial for detecting security threats and incidents.

Common Security Indicators

  • Port Scanning: Sequential connection attempts
  • DDoS Attacks: High volume of similar packets
  • Malware Communication: Unusual traffic patterns
  • Data Exfiltration: Large outbound transfers
  • Protocol Anomalies: Malformed or suspicious packets

Security Analysis Techniques

# Port Scan Detection:
- Multiple SYN packets to different ports
- Short time intervals between attempts
- No corresponding application data
- RST responses from closed ports

# Suspicious Traffic Patterns:
- Connections to known bad IPs
- Unusual protocols or ports
- Encrypted traffic to unexpected destinations
- Beaconing behavior (regular intervals)

# Malware Indicators:
- DNS queries to suspicious domains
- HTTP requests with unusual User-Agents
- Binary data in unexpected protocols
- Command and control communications

# Data Exfiltration Signs:
- Large file transfers
- Compressed or encrypted payloads
- Unusual upload patterns
- Off-hours data transfers

Practical Analysis Scenarios

Scenario 1: Slow Web Performance

# Investigation Steps:
1. Capture HTTP traffic to slow website
2. Analyze TCP handshake timing
3. Check for retransmissions
4. Examine HTTP response times
5. Look for DNS resolution delays

# Common Causes:
- High network latency
- Server processing delays
- Large content downloads
- DNS resolution issues
- Network congestion

Scenario 2: Intermittent Connectivity

# Investigation Steps:
1. Monitor connection establishment
2. Look for connection resets (RST)
3. Check for duplicate IP addresses
4. Analyze ARP traffic
5. Examine DHCP lease renewals

# Common Causes:
- IP address conflicts
- Faulty network hardware
- Intermittent cable issues
- DHCP server problems
- Wireless interference

Best Practices for Packet Analysis

Capture Best Practices

  • Use appropriate filters: Capture only relevant traffic
  • Consider storage space: Large captures consume disk space
  • Time synchronization: Ensure accurate timestamps
  • Multiple capture points: Get complete picture
  • Document methodology: Record analysis procedures

Analysis Best Practices

  • Start with overview: Understand traffic patterns first
  • Use statistics: Leverage built-in analysis tools
  • Follow conversations: Track complete sessions
  • Correlate events: Look for related activities
  • Validate findings: Confirm analysis with additional data
