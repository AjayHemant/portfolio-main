Common Network Attacks

Understand attack vectors and defensive countermeasures

Introduction to Network Attacks

Understanding common network attacks is crucial for cybersecurity professionals. This knowledge helps in implementing proper defenses and recognizing attack patterns in network traffic.

Attack Categories

  • Reconnaissance: Information gathering attacks
  • Access Attacks: Gaining unauthorized access
  • Denial of Service: Disrupting network services
  • Data Attacks: Stealing or manipulating data
  • Man-in-the-Middle: Intercepting communications

Reconnaissance Attacks

Reconnaissance is the first phase of most attacks, involving information gathering about targets.

Network Scanning

# Port Scanning Techniques:

# TCP Connect Scan
nmap -sT target.com
- Completes full TCP handshake
- Easily detected by firewalls
- Most reliable scan type

# SYN Scan (Stealth Scan)
nmap -sS target.com
- Sends SYN packets only
- Doesn't complete handshake
- Harder to detect

# UDP Scan
nmap -sU target.com
- Scans UDP ports
- Slower than TCP scans
- Often filtered by firewalls

# Common Port Scan Patterns:
Sequential: 1, 2, 3, 4, 5...
Random: 443, 22, 1433, 80, 3389...
Top Ports: 80, 443, 22, 21, 25...

OS Fingerprinting

# Operating System Detection Methods:

# Active Fingerprinting:
nmap -O target.com
- Sends crafted packets
- Analyzes responses
- Compares to signature database

# Passive Fingerprinting:
- Analyzes existing traffic
- TTL values
- Window sizes
- TCP options
- Packet fragmentation

# Common OS Signatures:
Windows: TTL=128, Window=65535
Linux: TTL=64, Window=5840
macOS: TTL=64, Window=65535

Service Enumeration

  • Banner Grabbing: Identifying service versions
  • SNMP Enumeration: Gathering system information
  • DNS Enumeration: Discovering subdomains and records
  • SMB Enumeration: Windows share information

Denial of Service (DoS) Attacks

DoS attacks aim to make network resources unavailable to legitimate users.

Volumetric Attacks

  • UDP Flood
  • ICMP Flood
  • DNS Amplification
  • NTP Amplification
  • Consume bandwidth

Protocol Attacks

  • SYN Flood
  • Ping of Death
  • Smurf Attack
  • Fragmentation Attacks
  • Exploit protocol weaknesses

Application Layer Attacks

  • HTTP Flood
  • Slowloris
  • RUDY Attack
  • DNS Query Flood
  • Target specific applications

SYN Flood Attack

# SYN Flood Attack Process:
1. Attacker sends multiple SYN packets
2. Server responds with SYN-ACK
3. Attacker never sends final ACK
4. Server keeps half-open connections
5. Connection table fills up
6. Legitimate connections rejected

# Attack Characteristics:
- High volume of SYN packets
- No corresponding ACK packets
- Source IPs often spoofed
- Targets TCP-based services

# Detection Indicators:
- Unusual SYN/ACK ratio
- High number of half-open connections
- Connection timeouts
- Service unavailability

DDoS (Distributed DoS)

  • Botnets: Networks of compromised computers
  • Amplification: Using third-party services to amplify attacks
  • Multi-vector: Combining different attack types
  • Harder to mitigate: Traffic from multiple sources

Man-in-the-Middle (MITM) Attacks

MITM attacks involve intercepting communications between two parties.

ARP Spoofing

# ARP Spoofing Process:
1. Attacker sends fake ARP responses
2. Associates attacker's MAC with victim's IP
3. Traffic intended for victim goes to attacker
4. Attacker can intercept, modify, or forward traffic

# Attack Tools:
ettercap -T -M arp:remote /192.168.1.1// /192.168.1.100//
arpspoof -i eth0 -t 192.168.1.100 192.168.1.1

# Detection Methods:
- Monitor ARP table changes
- Detect duplicate IP addresses
- Use static ARP entries
- Implement ARP inspection

DNS Spoofing

# DNS Spoofing Techniques:

# DNS Cache Poisoning:
- Inject false DNS records into cache
- Redirect users to malicious sites
- Exploit DNS resolver vulnerabilities

# DNS Hijacking:
- Compromise DNS server
- Modify DNS records
- Redirect entire domains

# Local DNS Spoofing:
- Modify local hosts file
- Use rogue DHCP server
- Provide false DNS servers

SSL/TLS Attacks

  • SSL Stripping: Downgrade HTTPS to HTTP
  • Certificate Spoofing: Use fake certificates
  • Session Hijacking: Steal session cookies
  • Weak Cipher Exploitation: Attack weak encryption

Password Attacks

Password attacks target authentication mechanisms to gain unauthorized access.

Brute Force

  • Try all possible combinations
  • Time-intensive but thorough
  • Effective against weak passwords
  • Can be distributed

Dictionary Attack

  • Use common password lists
  • Faster than brute force
  • Based on human behavior
  • Include variations and mutations

Rainbow Tables

  • Pre-computed hash tables
  • Trade space for time
  • Effective against unsalted hashes
  • Defeated by salt values

Password Attack Tools

# Common Password Attack Tools:

# Hydra (Network Login Cracker):
hydra -l admin -P passwords.txt ssh://target.com
hydra -L users.txt -P passwords.txt http-post-form://target.com

# John the Ripper (Hash Cracker):
john --wordlist=passwords.txt hashes.txt
john --rules --wordlist=passwords.txt hashes.txt

# Hashcat (GPU-accelerated):
hashcat -m 0 -a 0 hashes.txt passwords.txt
hashcat -m 1000 -a 3 hashes.txt ?a?a?a?a?a?a

# Medusa (Parallel Login Bruter):
medusa -h target.com -u admin -P passwords.txt -M ssh

Network Sniffing Attacks

Network sniffing involves capturing and analyzing network traffic to extract sensitive information.

Passive Sniffing

  • Hub-based Networks: All traffic visible to all ports
  • Wireless Networks: Radio waves can be intercepted
  • Promiscuous Mode: Capture all network traffic
  • Undetectable: No active network participation

Active Sniffing

# Active Sniffing Techniques:

# MAC Flooding:
- Overflow switch MAC address table
- Force switch into hub mode
- All traffic becomes visible

# ARP Spoofing:
- Redirect traffic through attacker
- Position as man-in-the-middle
- Intercept switched network traffic

# DHCP Spoofing:
- Provide false network configuration
- Redirect traffic through attacker
- Capture credentials and data

Sniffing Countermeasures

  • Encryption: Use HTTPS, SSH, VPNs
  • Switch Security: Port security, DHCP snooping
  • Network Segmentation: VLANs, subnets
  • Monitoring: Detect suspicious ARP activity

Social Engineering Attacks

Social engineering exploits human psychology rather than technical vulnerabilities.

Common Techniques

  • Phishing: Fraudulent emails requesting information
  • Pretexting: Creating false scenarios to gain trust
  • Baiting: Offering something to trigger curiosity
  • Tailgating: Following authorized personnel into secure areas
  • Quid Pro Quo: Offering services in exchange for information

Technical Social Engineering

# Technical Social Engineering Methods:

# Email Spoofing:
- Forge sender addresses
- Impersonate trusted entities
- Bypass email filters

# Website Cloning:
- Create fake login pages
- Harvest credentials
- Redirect to legitimate sites

# USB Drops:
- Leave infected USB devices
- Rely on human curiosity
- Autorun malicious payloads

# Fake Wi-Fi Hotspots:
- Create rogue access points
- Capture user credentials
- Monitor network traffic

Attack Detection and Prevention

Detection Strategies

  • Network Monitoring: Continuous traffic analysis
  • Intrusion Detection Systems: Automated threat detection
  • Log Analysis: Review system and network logs
  • Behavioral Analysis: Detect anomalous patterns
  • Threat Intelligence: Use external threat data

Prevention Measures

Network Security

  • Firewalls and ACLs
  • Network segmentation
  • VPN implementation
  • Intrusion prevention systems

Access Control

  • Strong authentication
  • Multi-factor authentication
  • Principle of least privilege
  • Regular access reviews

Security Awareness

  • Employee training
  • Phishing simulations
  • Security policies
  • Incident reporting

Incident Response

# Incident Response Process:

1. Preparation:
   - Develop response procedures
   - Train response team
   - Prepare tools and resources

2. Identification:
   - Detect security incidents
   - Analyze alerts and logs
   - Determine scope and impact

3. Containment:
   - Isolate affected systems
   - Prevent further damage
   - Preserve evidence

4. Eradication:
   - Remove threats
   - Patch vulnerabilities
   - Update security controls

5. Recovery:
   - Restore normal operations
   - Monitor for reoccurrence
   - Validate system integrity

6. Lessons Learned:
   - Document incident
   - Update procedures
   - Improve defenses
