OSI and TCP/IP Models

Deep dive into network layer models and protocol stacks

Introduction to Network Models

Network models provide a framework for understanding how data moves through networks. The two most important models are the OSI (Open Systems Interconnection) model and the TCP/IP model.

Why Network Models Matter

  • Standardization: Common framework for network communication
  • Troubleshooting: Systematic approach to problem-solving
  • Design: Guidelines for network architecture
  • Interoperability: Different systems can communicate
  • Education: Structured learning approach

The OSI Model (7 Layers)

The OSI model divides network communication into seven distinct layers, each with specific responsibilities.

Layer 7 - Application

User interface, network services (HTTP, FTP, SMTP)

Layer 6 - Presentation

Data formatting, encryption, compression

Layer 5 - Session

Session management, dialog control

Layer 4 - Transport

End-to-end delivery (TCP, UDP)

Layer 3 - Network

Routing, logical addressing (IP)

Layer 2 - Data Link

Frame formatting, error detection (Ethernet)

Layer 1 - Physical

Electrical signals, cables, hardware

OSI Layer Functions in Detail

# OSI Layer Breakdown:

Layer 7 (Application):
- Web browsers (HTTP/HTTPS)
- Email clients (SMTP, POP3, IMAP)
- File transfer (FTP, SFTP)
- Remote access (SSH, Telnet)

Layer 6 (Presentation):
- SSL/TLS encryption
- Data compression (ZIP, GZIP)
- Character encoding (ASCII, Unicode)
- Image formats (JPEG, PNG)

Layer 5 (Session):
- Session establishment/termination
- Authentication
- Authorization
- Session checkpointing

Layer 4 (Transport):
- TCP: Reliable, connection-oriented
- UDP: Fast, connectionless
- Port numbers (0-65535)
- Flow control, error recovery

Layer 3 (Network):
- IP addressing (IPv4, IPv6)
- Routing protocols (OSPF, BGP)
- ICMP (ping, traceroute)
- Packet forwarding

Layer 2 (Data Link):
- MAC addressing
- Frame formatting
- Error detection (CRC)
- Switch operations

Layer 1 (Physical):
- Electrical voltages
- Cable specifications
- Wireless frequencies
- Hub operations

The TCP/IP Model (4 Layers)

The TCP/IP model is the practical implementation used by the Internet. It has four layers that map to the OSI model.

Application Layer

Combines OSI layers 5, 6, 7 - User applications and services

Transport Layer

Same as OSI Layer 4 - TCP/UDP protocols

Internet Layer

Same as OSI Layer 3 - IP routing and addressing

Network Access Layer

Combines OSI layers 1, 2 - Physical and data link

TCP/IP vs OSI Comparison

# Layer Mapping:

TCP/IP Model          OSI Model
================      ==================
Application    <-->   Application (7)
               <-->   Presentation (6)
               <-->   Session (5)

Transport      <-->   Transport (4)

Internet       <-->   Network (3)

Network Access <-->   Data Link (2)
               <-->   Physical (1)

Data Encapsulation Process

As data moves down the layers, each layer adds its own header (and sometimes trailer) to the data.

# Data Encapsulation (Top to Bottom):

Application Layer:
[Data] ‚Üí User data (email, web page, file)

Transport Layer:
[TCP/UDP Header][Data] ‚Üí Segment/Datagram

Network Layer:
[IP Header][TCP/UDP Header][Data] ‚Üí Packet

Data Link Layer:
[Frame Header][IP Header][TCP/UDP Header][Data][Frame Trailer] ‚Üí Frame

Physical Layer:
Electrical signals, light pulses, radio waves ‚Üí Bits

# Data De-encapsulation (Bottom to Top):
The receiving device reverses this process,
removing headers at each layer until the
original data reaches the application.

Protocol Data Units (PDUs)

  • Application Layer: Data
  • Transport Layer: Segment (TCP) or Datagram (UDP)
  • Network Layer: Packet
  • Data Link Layer: Frame
  • Physical Layer: Bits

Common Protocols by Layer

Application Layer Protocols

  • HTTP/HTTPS: Web traffic (ports 80/443)
  • FTP: File transfer (ports 20/21)
  • SMTP: Email sending (port 25)
  • POP3/IMAP: Email retrieval (ports 110/143)
  • DNS: Domain name resolution (port 53)
  • DHCP: Dynamic IP assignment (ports 67/68)
  • SSH: Secure remote access (port 22)
  • Telnet: Insecure remote access (port 23)

Transport Layer Protocols

  • TCP: Reliable, connection-oriented
  • UDP: Fast, connectionless

Network Layer Protocols

  • IP: Internet Protocol (IPv4/IPv6)
  • ICMP: Internet Control Message Protocol
  • ARP: Address Resolution Protocol
  • OSPF: Open Shortest Path First
  • BGP: Border Gateway Protocol

TCP vs UDP Comparison

TCP (Transmission Control Protocol)

  • Connection-oriented
  • Reliable delivery
  • Error checking and correction
  • Flow control
  • Ordered delivery
  • Higher overhead

Use cases: Web browsing, email, file transfer

UDP (User Datagram Protocol)

  • Connectionless
  • Best-effort delivery
  • No error correction
  • No flow control
  • No delivery guarantee
  • Lower overhead

Use cases: DNS, DHCP, streaming, gaming

TCP Three-Way Handshake

# TCP Connection Establishment:

Client                    Server
  |                         |
  |-------- SYN ----------->|  (1) Client requests connection
  |                         |
  |<----- SYN-ACK ----------|  (2) Server acknowledges and responds
  |                         |
  |-------- ACK ----------->|  (3) Client acknowledges server response
  |                         |
  |    Connection Established
  |                         |
  |<===== Data Transfer ====>|
  |                         |

# TCP Connection Termination (Four-Way Handshake):
  |-------- FIN ----------->|  (1) Client initiates close
  |<------- ACK ------------|  (2) Server acknowledges
  |<------- FIN ------------|  (3) Server initiates close
  |-------- ACK ----------->|  (4) Client acknowledges

Practical Network Troubleshooting

Understanding the OSI model helps in systematic network troubleshooting.

Layer-by-Layer Troubleshooting Approach

# Troubleshooting from Bottom Up:

Layer 1 (Physical):
- Check cable connections
- Verify power to devices
- Check link lights/LEDs
- Test with known good cables

Layer 2 (Data Link):
- Check switch port status
- Verify VLAN configuration
- Check MAC address tables
- Look for duplex mismatches

Layer 3 (Network):
- Verify IP configuration
- Check routing tables
- Test with ping
- Verify subnet masks

Layer 4 (Transport):
- Check port availability
- Verify firewall rules
- Test with telnet to specific ports
- Check for port conflicts

Layer 5-7 (Upper Layers):
- Check application configuration
- Verify DNS resolution
- Test application connectivity
- Check authentication/authorization

Common Troubleshooting Commands

# Network Diagnostic Commands:

# Layer 1-2 Testing:
ipconfig /all          # Windows network config
ifconfig -a            # Linux/Mac network config
arp -a                 # ARP table

# Layer 3 Testing:
ping 8.8.8.8          # Test connectivity
tracert google.com     # Trace route (Windows)
traceroute google.com  # Trace route (Linux/Mac)

# Layer 4 Testing:
netstat -an           # Show all connections
telnet google.com 80  # Test specific port
nmap -p 80 target.com # Port scanning

# Layer 7 Testing:
nslookup google.com   # DNS resolution
curl -I google.com    # HTTP headers
wget google.com       # Download test
