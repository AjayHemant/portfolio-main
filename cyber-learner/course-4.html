Networking Fundamentals

Understand network architecture, protocols, and communication basics

Introduction to Computer Networks

Computer networks are interconnected systems that allow devices to communicate and share resources. Understanding networking is crucial for cybersecurity professionals.

What is a Network?

  • Definition: A collection of interconnected devices
  • Purpose: Share resources and communicate
  • Components: Nodes, links, protocols
  • Types: LAN, WAN, MAN, PAN
  • Topology: Physical and logical arrangement

Network Types and Topologies

Network Types by Size

PAN

Personal Area Network - Very small range (Bluetooth, USB)

LAN

Local Area Network - Building or campus (Ethernet, Wi-Fi)

MAN

Metropolitan Area Network - City-wide coverage

WAN

Wide Area Network - Large geographical areas (Internet)

Common Network Topologies

  • Bus: All devices connected to a single cable
  • Star: All devices connected to a central hub
  • Ring: Devices connected in a circular fashion
  • Mesh: Every device connected to every other device
  • Hybrid: Combination of multiple topologies

Network Hardware Components

Essential Network Devices

  • Router: Connects different networks and routes traffic
  • Switch: Connects devices within the same network
  • Hub: Basic connection point (largely obsolete)
  • Access Point: Provides wireless network access
  • Firewall: Filters network traffic for security
  • Modem: Converts digital signals for transmission

Network Interface Cards (NICs)

NICs are hardware components that allow devices to connect to networks:

  • Ethernet NIC: Wired network connections
  • Wireless NIC: Wi-Fi connections
  • MAC Address: Unique hardware identifier
  • Speed: 10/100/1000 Mbps or higher

Network Protocols Overview

Protocol Layers

Networks use layered protocols to manage communication:

  • Physical Layer: Electrical signals, cables
  • Data Link Layer: Frame formatting, error detection
  • Network Layer: Routing, IP addressing
  • Transport Layer: Reliable delivery (TCP/UDP)
  • Application Layer: User applications (HTTP, FTP)

Common Protocols

HTTP/HTTPS

Web traffic (ports 80/443)

FTP/SFTP

File transfer (ports 21/22)

SSH

Secure remote access (port 22)

DNS

Domain name resolution (port 53)

DHCP

Dynamic IP assignment (ports 67/68)

SMTP/POP3

Email services (ports 25/110)

Data Transmission Methods

Transmission Types

  • Simplex: One-way communication (radio broadcast)
  • Half-Duplex: Two-way, but not simultaneous (walkie-talkie)
  • Full-Duplex: Simultaneous two-way communication (telephone)

Switching Methods

  • Circuit Switching: Dedicated path for entire communication
  • Packet Switching: Data broken into packets, routed independently
  • Message Switching: Store-and-forward messaging

Network Security Basics

Common Security Threats

  • Eavesdropping: Unauthorized interception of data
  • Man-in-the-Middle: Attacker intercepts communication
  • Denial of Service: Overwhelming network resources
  • Spoofing: Impersonating legitimate network entities
  • Packet Sniffing: Capturing and analyzing network traffic

Basic Security Measures

  • Encryption: Protecting data in transit
  • Authentication: Verifying user identity
  • Access Control: Limiting network access
  • Firewalls: Filtering network traffic
  • VPNs: Secure remote connections

Network Troubleshooting Tools

Command Line Tools

# Ping - Test connectivity
ping google.com
ping 8.8.8.8

# Traceroute - Show path to destination
tracert google.com (Windows)
traceroute google.com (Linux/Mac)

# Netstat - Show network connections
netstat -an
netstat -rn (routing table)

# Nslookup - DNS queries
nslookup google.com
nslookup 8.8.8.8

# Ipconfig/ifconfig - Network configuration
ipconfig /all (Windows)
ifconfig (Linux/Mac)

Network Monitoring

  • Bandwidth Monitoring: Track network usage
  • Latency Testing: Measure response times
  • Packet Analysis: Examine network traffic
  • Performance Metrics: Monitor network health
