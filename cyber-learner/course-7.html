Subnetting & IP Math

Master IP addressing, CIDR notation, and network calculations

Introduction to Subnetting

Subnetting is the process of dividing a network into smaller sub-networks. It's essential for efficient IP address management, network security, and performance optimization.

Why Subnet Networks?

  • Efficient IP Usage: Reduce IP address waste
  • Network Security: Isolate network segments
  • Performance: Reduce broadcast domains
  • Organization: Logical network structure
  • Scalability: Easier network management

IP Address Structure

Understanding IP address structure is fundamental to subnetting.

# IPv4 Address Structure (32 bits):

# Default Subnet Masks:
Class A: 255.0.0.0     (/8)  - 8 network bits, 24 host bits

Subnetting Process

Step-by-step approach to subnet a network.

Example: Subnet 192.168.1.0/24 into 4 subnets

# Step 1: Determine required subnet bits
4 subnets = 2¬≤ subnets
Need 2 additional bits for subnetting
New prefix: /24 + 2 = /26

# Step 2: Calculate new subnet mask
/26 = 255.255.255.192

# Step 3: Calculate subnet increment
256 - 192 = 64 (increment)

# Step 4: List subnets
Subnet 1: 192.168.1.0/26   (192.168.1.0 - 192.168.1.63)
Subnet 2: 192.168.1.64/26  (192.168.1.64 - 192.168.1.127)
Subnet 3: 192.168.1.128/26 (192.168.1.128 - 192.168.1.191)
Subnet 4: 192.168.1.192/26 (192.168.1.192 - 192.168.1.255)

# Step 5: Identify key addresses for each subnet
Subnet 1 (192.168.1.0/26):
- Network Address: 192.168.1.0
- First Host: 192.168.1.1
- Last Host: 192.168.1.62
- Broadcast: 192.168.1.63

Variable Length Subnet Masking (VLSM)

VLSM allows different subnet sizes within the same network, optimizing IP address usage.

# VLSM Example: 192.168.1.0/24
Requirements:
- Subnet A: 100 hosts
- Subnet B: 50 hosts  
- Subnet C: 25 hosts
- Subnet D: 10 hosts

# Step 1: Order by size (largest first)
Subnet A: 100 hosts ‚Üí need /25 (126 hosts)
Subnet B: 50 hosts  ‚Üí need /26 (62 hosts)
Subnet C: 25 hosts  ‚Üí need /27 (30 hosts)
Subnet D: 10 hosts  ‚Üí need /28 (14 hosts)

# Step 2: Assign subnets
Subnet A: 192.168.1.0/25   (192.168.1.0 - 192.168.1.127)
Subnet B: 192.168.1.128/26 (192.168.1.128 - 192.168.1.191)
Subnet C: 192.168.1.192/27 (192.168.1.192 - 192.168.1.223)
Subnet D: 192.168.1.224/28 (192.168.1.224 - 192.168.1.239)

# Remaining: 192.168.1.240/28 - 192.168.1.255/28 (available)

Practical Subnetting Examples

Example 1: Office Network

# Scenario: 192.168.10.0/24 for office with 4 departments
# Each department needs 50 hosts

# Solution: Use /26 subnets (62 hosts each)
