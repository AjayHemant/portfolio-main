Protocols Overview

Explore HTTP, HTTPS, FTP, SSH, and other essential protocols

Introduction to Network Protocols

Network protocols are standardized rules that define how devices communicate over networks. Understanding these protocols is crucial for cybersecurity professionals to identify vulnerabilities and secure communications.

What are Network Protocols?

  • Definition: Set of rules governing network communication
  • Purpose: Ensure reliable and standardized data exchange
  • Components: Syntax, semantics, and timing
  • Standards: Defined by organizations like IETF, IEEE
  • Security: Some protocols include built-in security features

HTTP and HTTPS

HyperText Transfer Protocol (HTTP) and its secure version (HTTPS) are fundamental to web communication.

HTTP (HyperText Transfer Protocol)

HTTP Basics

  • Port: 80
  • Stateless protocol
  • Request-response model
  • Plain text transmission
  • No built-in security

HTTPS (HTTP Secure)

  • Port: 443
  • Encrypted with SSL/TLS
  • Authentication support
  • Data integrity protection
  • Required for sensitive data

HTTP Methods

# Common HTTP Methods:

GET     - Retrieve data from server
POST    - Send data to server
PUT     - Update/replace resource
DELETE  - Remove resource
HEAD    - Get headers only
OPTIONS - Get allowed methods
PATCH   - Partial update

# HTTP Request Example:
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
Connection: keep-alive

# HTTP Response Example:
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Server: Apache/2.4.41

<html>...</html>

HTTP Status Codes

  • 1xx Informational: 100 Continue, 101 Switching Protocols
  • 2xx Success: 200 OK, 201 Created, 204 No Content
  • 3xx Redirection: 301 Moved Permanently, 302 Found, 304 Not Modified
  • 4xx Client Error: 400 Bad Request, 401 Unauthorized, 404 Not Found
  • 5xx Server Error: 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable

FTP and SFTP

File Transfer Protocol (FTP) and its secure variants are used for transferring files between systems.

FTP (File Transfer Protocol)

# FTP Characteristics:
- Port 21 (control), Port 20 (data)
- Two separate connections
- Active vs Passive modes
- Plain text authentication
- No encryption by default

# FTP Commands:
USER username    - Authenticate user
PASS password    - Provide password
LIST            - List directory contents
RETR filename   - Download file
STOR filename   - Upload file
PWD             - Print working directory
CWD directory   - Change directory
QUIT            - Close connection

# FTP Session Example:
220 Welcome to FTP server
USER alice
331 Password required
PASS secret123
230 Login successful
PWD
257 "/" is current directory
LIST
150 Opening data connection
226 Transfer complete

Secure File Transfer Protocols

  • SFTP: SSH File Transfer Protocol (port 22)
  • FTPS: FTP over SSL/TLS (ports 989/990)
  • SCP: Secure Copy Protocol (port 22)

SSH (Secure Shell)

SSH provides secure remote access and command execution over encrypted connections.

# SSH Features:
- Port 22
- Strong encryption
- Public key authentication
- Port forwarding/tunneling
- X11 forwarding
- File transfer capabilities

# SSH Connection:
ssh username@hostname
ssh -p 2222 user@server.com    # Custom port
ssh -i keyfile user@server     # Private key auth

# SSH Key Generation:
ssh-keygen -t rsa -b 4096
ssh-keygen -t ed25519

# SSH Configuration (~/.ssh/config):
Host myserver
    HostName server.example.com
    User alice
    Port 2222
    IdentityFile ~/.ssh/id_rsa

# SSH Tunneling:
ssh -L 8080:localhost:80 user@server    # Local port forwarding
ssh -R 8080:localhost:80 user@server    # Remote port forwarding
ssh -D 1080 user@server                 # SOCKS proxy

SSH Security Best Practices

  • Disable root login: PermitRootLogin no
  • Use key-based authentication: Disable password auth
  • Change default port: Use non-standard port
  • Limit user access: AllowUsers directive
  • Enable fail2ban: Block brute force attempts

PTR     - Reverse lookup
TXT     - Text records
SOA     - Start of authority

# DNS Query Process:
1. Client queries local DNS resolver
2. Resolver queries root name server
3. Root server responds with TLD server
4. Resolver queries TLD server
5. TLD server responds with authoritative server
6. Resolver queries authoritative server
7. Authoritative server responds with IP
8. Resolver caches and returns result

# DNS Tools:
nslookup google.com
dig google.com
host google.com
dig @8.8.8.8 google.com MX

DNS Security Concerns

  • DNS Spoofing: Fake DNS responses
  • Cache Poisoning: Corrupt DNS cache
  • DNS Tunneling: Data exfiltration via DNS
  • DDoS Amplification: Using DNS for attacks
  • DNSSEC: DNS Security Extensions for validation

DHCP (Dynamic Host Configuration Protocol)

DHCP automatically assigns IP addresses and network configuration to devices.

# DHCP Process (DORA):
1. DISCOVER  - Client broadcasts request
2. OFFER     - Server offers IP address
3. REQUEST   - Client requests offered IP
4. ACK       - Server acknowledges assignment

# DHCP Configuration Options:
- IP address and subnet mask
- Default gateway
- DNS servers
- Domain name
- Lease time
- NTP servers
- TFTP server (for PXE boot)

# DHCP Ports:
Server: UDP 67
Client: UDP 68

# DHCP Lease States:
INIT        - Initial state
SELECTING   - Waiting for offers
REQUESTING  - Requesting specific IP
BOUND       - IP assigned and active
RENEWING    - Attempting to renew lease
REBINDING   - Attempting to rebind lease
INIT-REBOOT - Reboot with previous IP

DHCP Security Issues

  • Rogue DHCP Servers: Unauthorized DHCP servers
  • DHCP Starvation: Exhausting IP pool
  • DHCP Spoofing: Fake DHCP responses
  • Man-in-the-Middle: Malicious gateway assignment

Email Protocols

Several protocols handle different aspects of email communication.

SMTP (Simple Mail Transfer Protocol)

  • Port: 25 (plain), 587 (submission), 465 (SSL)
  • Sends outgoing mail
  • Server-to-server transfer
  • Text-based protocol

POP3 (Post Office Protocol v3)

  • Port: 110 (plain), 995 (SSL)
  • Downloads mail to client
  • Deletes from server
  • Single device access

IMAP (Internet Message Access Protocol)

  • Port: 143 (plain), 993 (SSL)
  • Syncs mail across devices
  • Server-side storage
  • Multi-device access

Other Important Protocols

SNMP (Simple Network Management Protocol)

  • Port: 161 (agent), 162 (trap)
  • Purpose: Network device management
  • Versions: SNMPv1, SNMPv2c, SNMPv3
  • Security: Community strings, encryption (v3)

NTP (Network Time Protocol)

  • Port: 123 (UDP)
  • Purpose: Time synchronization
  • Accuracy: Millisecond precision
  • Hierarchy: Stratum levels (0-15)

LDAP (Lightweight Directory Access Protocol)

  • Port: 389 (plain), 636 (SSL)
  • Purpose: Directory services
  • Use cases: Authentication, user management
  • Structure: Hierarchical directory tree

Protocol Security Analysis

Secure vs Insecure Protocols

Insecure Protocols

  • HTTP (port 80)
  • FTP (ports 20/21)
  • Telnet (port 23)
  • SMTP (port 25)
  • POP3 (port 110)
  • IMAP (port 143)
  • SNMP v1/v2c

Secure Alternatives

  • HTTPS (port 443)
  • SFTP/FTPS (ports 22/990)
  • SSH (port 22)
  • SMTPS (port 465/587)
  • POP3S (port 995)
  • IMAPS (port 993)
  • SNMP v3

Common Protocol Vulnerabilities

  • Plain text transmission: Credentials and data exposed
  • Weak authentication: Default or weak passwords
  • Protocol downgrade: Forcing use of weaker versions
  • Man-in-the-middle: Intercepting communications
  • Replay attacks: Reusing captured authentication
